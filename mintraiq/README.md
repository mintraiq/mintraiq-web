# MintrAIQ — static React embed

Production build output lives in **`react-embed/`** (generated by `npm run build:react` from `apps/ninja-react/`).

| File | Role |
|------|------|
| `react-embed/ninja-ui.js` | Vite bundle: budget planner, forecast, weekly planner (`data-ninja-page` on `#ninja-react-root`). |

Host pages set `<meta name="mintraiq-samples-base" content="…">` pointing at `docs/samples/`, then load this script as `type="module"`.

---

## Iframe status (repo-wide)

| Area | Uses `<iframe>`? |
|------|------------------|
| **`web/*.html`**, **`portal/*.html`**, **`legacy/pages/*.html`** | **No** — there are no `<iframe>` tags in these static pages. |
| **React views** (budget / forecast / weekly) | **No** — they mount into a `div` and fetch JSON. |
| **`legacy/templates/*.html` (Jinja)** | May still describe old embed patterns; not used by the static Vercel bundle unless you render Jinja server-side. |
| **`.iframe-panel` in `shell.css`** | **Unused** — kept for backwards compatibility; safe to delete later. |

Previously, **budget** and **forecast** used a Flask HTML iframe (`iframe-flask.js`); that path was removed in favour of this bundle.
